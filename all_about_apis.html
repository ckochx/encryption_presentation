What is API?

Well-defined and (hopefully) well documented structured data

Share data between disparate computer systems

Expose functionality in a limited and controlled* manner

*Results will vary

    • Super. How does an API work?

    Short answer: HTTP Verbs (GET, POST, PUT, DELETE, etc)

    Longer answer:

    REST

    representational state transfer

    Not an official protocol, more of an architectural arrangement.

    Using HTTP verbs in a RESTful manner permits a fire-and-forget approach to data sharing.

    And What does it look like

    Using Frontrush as an example:

    Get some athlete data:

    [ { "General":{ "FirstName":"Wilson", "LastName":"Phillips", "EmailAddress":"wphillips@recruityo.com", "CellPhoneNumber":null, "ContactNumber":"123.456.789", "Address1":"178 Stafford Way", "City":"Princeton", "State":"NJ", "Zip":"08822", "GraduationYear":2018 }, "Custom":{ "LogoPath":"/your_logo.png", "Site Name":"Your Site", "Grade":"", "High School":"Princeton High School", "ACT":"", "SAT":"0", "GPA":"85", "Position":"Goalie", "Jersey Number":"3", "Height":"5'6\"", "PartnerPlayerID":"yoursite222", "PartnerID":"99972010", "DOB":"02/09/1999" } } ]

    Set it to a variable named athlete_data

    Build a URL

    url = https://frontrush.com/FRConnectServices/athlete/recruit/send/profile?oauth_consumer_key=my_secret_key&oauth_token=user_session_token

    Use the correct HTTP method: POST

    Set the body to equal the athlete_data variable and POST it to the receiving service (Frontrush)

    You're (mostly) done!

    The receiving service will do a number of things prior to and after receiving the data you send.

    They may reject and ignore the entire request, if you did not include the right authentication, or encryption, or body formatting

    They may process what you sent.

    They service should alert you with a well-formatted message that tells you what happened to your request.

    Your sending service can listen for the responses and respond with additional actions.

    OR

    Your sending service can ignore any responses and keep doing other things.

    There are a number of ways to secure API communications.

    Which we won't bother talking about here, unless anyone really wants to.

    How does NCSA use APIs?
    Message Center:

    Handles (most) email sending and receiving
    Stores all the messages
    Stores the read receipts
    Can tell you about message counts (read, sent, received, spam, etc)

    Athlete Profile Service

    Is an interface to the database for client (athlete) data
    Can both read and write data (BeRecruited leads-- at one point)

    Frontrush Service

    Listens for calls from Coach RMS and Client RMS to send an/many athlete/athletess to Frontrush
    Sends our Athlete Data in a format that Frontrush can consume
    Relies upon Athlete Profile Service to get client data
    Listens for requests from Frontrush to update athlete data (To keep Frontrush data fresh)
